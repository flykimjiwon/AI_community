PHILOSOPHY · 세 개의 축

꿀은 빨리 나지 않는다.

하니모는 "빠르게 만들고 비싸게 파는" AI 툴을 만들지 않습니다. 천천히 쌓이고, 당신 기계에 얹히고, 필요할 때 꺼내 쓰는 형태를 지향합니다.

01 / OPEN
열린 코드
전부 MIT. 하니모의 어떤 부분도 유료 tier 뒤에 숨어 있지 않습니다. 포크하고, 베끼고, 팔고, 다시 안 줘도 됩니다 — 그게 오픈소스의 규칙.
02 / WARM
따뜻한 기본값
프롬프트는 짧고, 팔레트는 꿀빛이고, 경고는 위협하지 않습니다. 터미널을 오래 열어두고 싶게 만드는 게 기본 설정의 목표입니다.
03 / COMPOSABLE
붙여 쓰는 구조
code가 rag를 부르고, rag가 webui 안에 뜨고, webui 결과가 community에 링크됩니다. 하나만 써도 되고, 넷 다 묶어도 됩니다. 잠금 효과 없음.

한 줄이면 됩니다. Ollama 있으면 API 키도 필요 없습니다.
